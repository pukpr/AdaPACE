The **P4 Session Control app** `p4` enables control of several target processes from the host. This could also serve as a runtime engine of a scripted interface

- Copy a session profile template from an example dir to the working directory.

- Each executing process is given an ordinal in the range P = 1 .. N where N is the total number of executables. Here P is an integer indicating a user-set Process ID (PID).

| `proc` | `( P,` | `"process",` | `"dir",` | `env("PRINTER=frenv32 ..."),` | `arg("rtcoe_system.cfg  ...") ).` |
|--------|--------|--------------|----------|-------------------------------|----------------------------------|
|        | *PID*  | *process-name* | *view* | *environment variables* | *command line arguments*         |

- Each process location is described by a line within the session profile. The **P** here matches to the Process ID in the `proc` declaration, same as PACE_NODE.

| `run` | `( P,` | `"target",` | `"P4 is ready",` | `"localhost",` | `trace(false) ).` |
|-------|--------|--------------|------------------|----------------|-------------------|
|       | *PID*  | *processor-name* | *up-string* | *DISPLAY-var* | *extra debugging trace* |

- Processes are released in groups; each group G = 1 .. K is described by a line within the session profile. Here G is an integer indicating a Group ID (GID).

| `group` | `( G,` | `from(P1),` | `to(P2),` | `wait(RelativeTime) ).` |
|---------|--------|-------------|-----------|-------------------------|
|         | *GID*  | *P1 = first*| *P2 = last* | *Launch at Time From Previous Group = XX . YY secs* |

- Process Peek/Poke Symbols are defined in the executable if they are globally visible, as in a GDB debugger

| `symvar` | `("VarName",` | `"process",` | `"QualifiedAdaName",` | `type ).` |
|----------|---------------|--------------|-----------------------|-----------|
|          | *Keyed Name*  | *process-name* | *Ada Name (with dot)* | *int, flt, char, bool, double, LengthOfString, or ["enum1", "enum2", .."]* |

    
For example, the following will start an X app in the background; the arguments become the **actual** launched application:

    ```sh
    proc (21,"generic_p4_launcher","dir", 
        env("PORT=1111"),arg ("/usr/bin/X11/xmessage hello")).
    ```
    Everything will work as normal, but you will lose peek/poke.

- Processes can be 3-bit color coded on output; the pattern is to place a color declaration in the `session.pro` file. The foreground and background color can be independently set, so 56 colors are possible:
    ```prolog
    color (1, black, white).
    color (2, red, white).
    color (3, green, white).
    color (4, yellow, white).
    color (5, blue, white).
    color (6, magenta, white).
    color (_, cyan, white). % the default color
    ```
