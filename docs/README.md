# docs

Design documents, architecture guides, and reference material for AdaPACE.

## Contents

| File / Directory | Description |
|---|---|
| `architecture.md` | High-level system architecture and design decisions |
| `comms-protocol.md` | IPC command-pattern protocol specification |
| `getting-started.md` | Step-by-step guide for new contributors |
| `api/` | Per-package API reference (generated from `src/`) |

## Contributing documentation

* Write in Markdown; use fenced code blocks with the `ada` language tag for Ada snippets.
* Place diagrams in `media/diagrams/` and reference them with relative links.
* Keep documents focused; prefer multiple short documents over one large one.
